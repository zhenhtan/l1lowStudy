# 实现重点特性

## 目的
该目录提供一个最小可运行的 POSIX TCP 回环通信示例（客户端/服务端）。

## 关键实现点
- 服务端流程：`socket -> setsockopt -> bind -> listen -> accept`。
- 客户端流程：`socket -> inet_pton -> connect -> send -> recv`。
- 使用请求-响应模型完成一次完整消息往返。
- 每个系统调用均带错误处理与 fd 关闭逻辑。
- 客户端与服务端分离为两个可执行程序，职责清晰。

## 实践价值
- 适合作为后续扩展并发连接、协议编解码、超时控制的基础模板。
