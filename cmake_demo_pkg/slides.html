两个版本的对比

cmake_demo（原版，保留不变）

  • consumer 手写 include 路径
  • consumer 手写 .a 文件绝对路径
  • 没有导出 CMake package 元数据
target_include_directories(consumer_demo PRIVATE
  "$HOME/.local/include"
)
target_link_libraries(consumer_demo PRIVATE
  "$HOME/.local/lib/libmath_lib.a"
)

cmake_demo_pkg（新版）

  • consumer 仅用 find_package
  • include / lib 由 imported target 自动传播
  • library 工程导出完整 package 元数据
find_package(MathLib REQUIRED CONFIG)

target_link_libraries(consumer_demo PRIVATE
  MathLib::math_lib
)

原版的问题

  • 路径硬编码，可移植性差
    换机器或换安装前缀就要改 CMakeLists
  • include 与 lib 分开管理，容易不一致
    头文件版本与静态库版本可能来自不同地方
  • 没有版本控制
    无法用 find_package(MathLib 1.0 REQUIRED) 做版本约束
  • 用法要求不自动传播
    编译选项、头文件目录都要手工在 consumer 侧重复写

解决方案：标准 CMake Package

库工程安装时，额外导出三份 CMake 文件，让 consumer 通过 find_package 发现并使用：

cmake --install
→
MathLibConfig.cmake
MathLibConfigVersion.cmake
MathLibTargets.cmake
find_package(MathLib)
→
MathLib::math_lib
→
include 自动传播
lib 自动传播

MathLibConfig.cmake

包的入口文件，include MathLibTargets.cmake

MathLibConfigVersion.cmake

版本兼容性检查，支持 SameMajorVersion 策略

MathLibTargets.cmake

定义 MathLib::math_lib imported target 及其属性

BUILD_INTERFACE vs INSTALL_INTERFACE

BUILD_INTERFACE

只在源码构建阶段生效（build tree 内）。

target_include_directories(math_lib PUBLIC
  $<BUILD_INTERFACE:${CMAKE_CURRENT_SOURCE_DIR}>
)

此时 math_lib 的 consumer 在同一源码树中，直接用源码目录下的头文件。

INSTALL_INTERFACE

只在 cmake --install 安装后、外部工程 find_package 消费时生效。

target_include_directories(math_lib PUBLIC
  $<INSTALL_INTERFACE:${CMAKE_INSTALL_INCLUDEDIR}>
)

此时头文件已被安装到 include/，imported target 带着安装后路径。

为什么原版注释掉了 INSTALL_INTERFACE 也能运行？

原版 consumer 直接链接裸 .a 文件，不是通过 imported target，CMake 不会从 math_lib 传播 INSTALL_INTERFACE。但在 find_package 模式下，缺少它会导致 consumer 找不到头文件。

库工程改动（Producer）

▲ cmake_demo / CMakeLists.txt
project(CmakeDemo)

include(GNUInstallDirs)

target_include_directories(math_lib PUBLIC
    # $<INSTALL_INTERFACE:...> 被注释掉
  $<BUILD_INTERFACE:${CMAKE_CURRENT_SOURCE_DIR}>
)

install(TARGETS math_lib demo
  RUNTIME DESTINATION bin
  ...
)
✔ cmake_demo_pkg / CMakeLists.txt
project(CmakeDemo VERSION 1.0.0)

include(GNUInstallDirs)
include(CMakePackageConfigHelpers)

target_include_directories(math_lib PUBLIC
  $<BUILD_INTERFACE:${CMAKE_CURRENT_SOURCE_DIR}>
    $<INSTALL_INTERFACE:${CMAKE_INSTALL_INCLUDEDIR}>
)

add_library(MathLib::math_lib ALIAS math_lib)

install(TARGETS math_lib demo
    EXPORT MathLibTargets
  RUNTIME DESTINATION bin
  ...
)
install(EXPORT MathLibTargets FILE ...)
configure_package_config_file(...)
write_basic_package_version_file(...)

消费方改动（Consumer）

▲ 原版：手写 include / lib 路径
set(MATH_LIB_ROOT "$ENV{HOME}/.local")

add_executable(consumer_demo src/main.cpp)

target_include_directories(consumer_demo PRIVATE
  "${MATH_LIB_ROOT}/include"
)
target_link_libraries(consumer_demo PRIVATE
  "${MATH_LIB_ROOT}/lib/libmath_lib.a"
)
✔ 新版：find_package + 命名空间 target
list(PREPEND CMAKE_PREFIX_PATH "$ENV{HOME}/.local")
find_package(MathLib REQUIRED CONFIG)

add_executable(consumer_demo src/main.cpp)

# include 路径由 imported target 自动传播，无需显式写
target_link_libraries(consumer_demo PRIVATE
  MathLib::math_lib
)

两条命令完成全流程

在 cmake_demo_pkg/ 目录下执行：

第一条：构建并安装库

# 配置 → 编译 → 安装
cmake -S . -B build_pkg \
  -DCMAKE_INSTALL_PREFIX=$HOME/.local/cmake_demo_pkg_install \
&& cmake --build build_pkg -j \
&& cmake --install build_pkg

第二条：构建并运行 consumer

# 配置 → 编译 → 运行
cmake -S consumer_project -B consumer_project/build \
  -DCMAKE_PREFIX_PATH=$HOME/.local/cmake_demo_pkg_install \
&& cmake --build consumer_project/build -j \
&& ./consumer_project/build/consumer_demo

find_package 如何找到 MathLib

  • CMake 在 CMAKE_PREFIX_PATH 下查找 lib/cmake/MathLib/MathLibConfig.cmake
  • MathLibConfig.cmake include MathLibTargets.cmake，定义 MathLib::math_lib
  • MathLib::math_lib 携带 include 路径和 lib 路径，链接时自动传播

总结

Producer 端

  • 增加 VERSION
  • 增加 INSTALL_INTERFACE
  • install(EXPORT ...)
  • generate Config + Version

Consumer 端

  • 删除手写 include 路径
  • 删除手写 .a 路径
  • find_package(MathLib CONFIG)
  • target_link_libraries(... MathLib::math_lib)

收益

  • 零硬编码路径
  • 支持版本约束
  • 用法自动传播
  • 跨平台 / 跨前缀可移植
