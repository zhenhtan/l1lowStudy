# 实现重点特性

## 目的
该目录聚焦并发编程中的典型协作模型与死锁诊断方法。

## 关键实现点
- `main.cpp` 中实现了经典生产者-消费者模型：`queue + mutex + condition_variable`。
- 使用原子变量统计处理进度与结果，避免竞争条件。
- 通过 `TrackedMutex + LockRegistry` 记录“线程等待哪把锁、哪把锁归谁持有”。
- 在快照基础上构建 wait-for 图并做环检测，识别死锁循环。
- `DeadlockMonitor` 以周期线程方式持续检测并输出报告。
- 附带故意反序加锁场景，构造可复现死锁用于验证检测链路。
- CMake 中启用 ThreadSanitizer 以增强并发问题定位能力。

## 实践价值
- 不仅展示“如何写并发”，还展示“如何观测并发故障”。
